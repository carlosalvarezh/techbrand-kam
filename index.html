Sincronización manual

Los datos se sincronizan automáticamente al guardar. Usa estos botones si necesitas forzar una sincronización.

Instrucciones de setup — una sola vez
Paso 1 — Crear el repositorio

En GitHub, crea un repositorio privado (recomendado) con cualquier nombre, ej. techbrand-kam. Márcalo como privado para que los datos no sean públicos.

Paso 2 — Subir el archivo HTML

Desde la terminal:

git init git add index.html git commit -m "KAM Tool v9" git branch -M main git remote add origin https://github.com/TU_USUARIO/TU_REPO.git git push -u origin main
Paso 3 — Crear el archivo data.json inicial

Crea un archivo data.json con el contenido mínimo:

{"empresas":[],"alertas":[],"ferias":[],"pipeline":[],"oportunidades":[],"catalogo":[]}

Si ya tienes datos en el computador, usa el botón Subir datos a GitHub ahora después de configurar el token — eso crea el archivo automáticamente con tus datos actuales.

Paso 4 — Habilitar GitHub Pages

En el repositorio → Settings → Pages → Source: Deploy from a branch → Branch: main → folder: / (root) → Save. En 1-2 minutos la app estará en https://TU_USUARIO.github.io/TU_REPO/

Paso 5 — Crear el Personal Access Token

GitHub → Settings → Developer settings → Personal access tokens → Fine-grained tokens → Generate new token:

  • Token name: KAM Tool sync
  • Expiration: 90 days (renovar cada 3 meses)
  • Repository access: Only select repositories → tu repo
  • Permissions → Repository permissions → Contents: Read and write

Copia el token generado (ghp_...) y pégalo en el campo de arriba. Solo se muestra una vez.

Paso 6 — Configurar en cada dispositivo

Abre la URL de GitHub Pages en el celular → panel Sincronización → ingresa los mismos datos (usuario, repo, token) → Guardar → Descargar datos desde GitHub. Listo.
