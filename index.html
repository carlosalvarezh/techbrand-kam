Modo de operación: cada vez que guardas un dato (empresa, oportunidad, actividad), la app guarda primero en localStorage y luego hace push automático a GitHub. Al abrir la app en un dispositivo nuevo, carga los datos desde GitHub automáticamente si el token está configurado.
Instrucciones de configuración inicial
Paso 1 — Crear el repositorio en GitHub
  1. Ve a github.com/new
  2. Nombre: techbrand-kam (o el que prefieras)
  3. Visibilidad: Private (recomendado — contiene datos comerciales)
  4. Marca "Add a README file" para que la rama main exista desde el inicio
  5. Crea el repositorio
  6. Sube el archivo data.json que descargaste: botón "Add file → Upload files"
  7. Sube index.html (este archivo, renombrado)
Paso 2 — Activar GitHub Pages
  1. En el repo: Settings → Pages
  2. Source: Deploy from a branch
  3. Branch: main / / (root)
  4. Guarda — en ~2 minutos tendrás una URL tipo https://tu-usuario.github.io/techbrand-kam/
  5. Abre esa URL en cualquier dispositivo — esa es tu app
Paso 3 — Crear el Personal Access Token
  1. Ve a github.com/settings/personal-access-tokens/new
  2. Tipo: Fine-grained token
  3. Nombre: KAM Tool Techbrand
  4. Expiration: No expiration o 1 año
  5. Repository access: Only select repositories → selecciona techbrand-kam
  6. Permissions → Repository permissions → Contents: Read and write
  7. Genera el token y cópialo — solo se muestra una vez
  8. Pégalo en el campo "Personal Access Token" de arriba y guarda la configuración
⚠
El token se guarda en el localStorage de este dispositivo. Si limpias el navegador o usas otro dispositivo, debes volver a ingresarlo. El token nunca está en el código fuente del HTML publicado.
